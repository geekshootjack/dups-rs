# dups

在文件名后附加 xxHash3-64 哈希值，确保任意两个文件都不可能因为相同的文件名而产生碰撞。相同的文件名必然对应相同的内容。

在后期制作的数据管理、备份和素材分发中，经常遇到**文件名碰撞**的问题：内容不同的多个文件却拥有相同的文件名。这导致两个严重后果：

- 素材相互覆盖，造成数据丢失
- 后期软件链接到错误的素材

我们的做法是在文件名末尾附加 16 个字符的 xxHash3-64 哈希值。这个哈希值直接源自文件内容，因此保证了：

- **内容不同** → 哈希不同 → 文件名不同 → 链接不会指向错误的素材
- **内容相同** → 哈希相同 → 文件名相同 → 真正的重复文件，链到哪条都无所谓

## 功能特性

- **跨平台** — Windows、macOS (Intel & ARM)、Linux
- **安全默认** — 执行前预览预演模式效果
- **防崩溃** — 预写日志，可通过 `undo` 恢复
- **幂等性** — 安全重新运行，已加后缀的文件会被跳过
- **灵活** — 支持使用已有的 `.xxh3` 清单或实时验证哈希
- **所有文件类型** — 默认处理视频，使用 `--all-files` 处理所有类型

## 安装

### 下载预编译二进制文件

从 [GitHub Releases](https://github.com/geekshootjack/dups-rs/releases) 获取最新版本

- `dups-windows-x86_64.exe` — Windows
- `dups-macos-x86_64` — macOS Intel
- `dups-macos-aarch64` — macOS Apple Silicon
- `dups-linux-x86_64` — Linux

### 从源代码编译

```bash
cd dups-rs
cargo build --release
./target/release/dups --help
```

## 使用方法

### 基本工作流

```bash
# 1. 预览要重命名的文件
dups /path/to/media

# 2. 如果效果满意，执行
dups /path/to/media --apply

# 3. 如需撤销
dups undo dups-applied-XXXXXX.csv
```

### 选项

- `<PATH>` — 包含媒体文件和 .xxh3 清单的目录
- `--apply` — 实际重命名文件（默认为预演模式预览）
- `--hashfile <PATH>` — 明确指定清单路径
- `--verify` — 重命名前验证哈希（较慢）
- `--all-files` — 重命名所有文件类型，不仅限视频
- `--undo <LOG>` — 撤销之前的重命名操作

## 工作原理

1. 扫描目录中 TeraCopy 兼容的 `.xxh3` 哈希文件
2. 读取每个文件的 xxHash3-64 校验和
3. 规划重命名：`原文件名.ext` → `原文件名_校验和.ext`
4. 检测冲突、幂等操作、缺失文件
5. 预演模式模式：显示预览；--apply 模式：使用防崩溃日志执行
6. 通过 CSV 日志撤销：读取实际重命名，安全地恢复

## 安全特性

- **默认预演模式** — 除非添加 `--apply`，否则不改动任何文件
- **幂等性** — 已加后缀的文件被跳过，安全重新运行
- **冲突检测** — 拒绝覆盖已存在的目标名称
- **防崩溃日志** — 每个重命名意图在执行前进行 fsync 操作
- **撤销跟踪** — 只恢复实际重命名的文件（磁盘状态感知）

```
$ dups /media/videos

======================================================================
摘要 / Summary
----------------------------------------------------------------------
  将重命名                    12
  已就绪(跳过)                 3
  清单有、磁盘无               2

日志已写出: dups-dryrun-20260630-153526.csv

*** 预演模式 (DRY-RUN) *** 未改动任何文件。
计划改名 12 个。确认无误后加 --apply 执行。
```

使用 `--apply` 执行：

```
$ dups /media/videos --apply

开始执行 12 个重命名...
  [OK] video1.mp4
  [OK] video2.mp4
  ...

重命名完成: 成功 12, 失败 0

======================================================================
执行完成
----------------------------------------------------------------------
[OK] 成功改名: 12 个
======================================================================

后续步骤：
  1. 检查结果是否满意
  2. 如需撤销所有改名，运行:
     dups undo dups-applied-20260630-153526.csv
  3. 详细日志见: dups-applied-20260630-153526.csv
```
